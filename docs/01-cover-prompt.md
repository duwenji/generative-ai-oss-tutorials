# コンポジション
テキストコンポジション:
- タイトルは上部20%ゾーンに配置し、カバーの構造的コアとして機能する。
- サブタイトルはタイトルのすぐ下に、2行で学習の複合性を表現。
- テキストはセンター～左寄せで技術的な整然さを演出。
- 9つのカテゴリ・11のツール群への学習パスを視覚的に暗示する配置。

ビジュアルコンポジション:
- メインビジュアル: 9つのノードまたはレイヤー（エージェント、RAG、推論基盤、UI、評価、マルチモーダル、可視化、プロトコル、コード生成）を表現する同心円または組織図的な相互接続構造。
- 中央エリア: 複数のノード（丸、六角形、矩形）を接続する流線型のエッジ（矢印・パイプライン）。データフローと技術要素の統合性を示す。
- アクセント: ネオン風のハイライト色（鮮やかな青、マゼンタ、シアン）がノード間のパスをアクセントする。
- 背景: 淡いグリッド、デジタルノイズパターン、またはシンプルな方眼背景。テック系のコンテキストを強化。
- デコレーション: 「Agent」「RAG」「LLM」「Multimodal」など主要キーワードを極小テキストで各ノード近傍に配置。
- レイアウト: 上部25%をテキスト、下部75%をノード・相互接続・データフロー図に割り当て。

カラースキーム:

カラー制約: カラー値（#hex）とカラー名はレンダリングガイダンスのみです — カラー名、16進コード、パレットラベルを画像内の可視テキストとして表示しないでください。

レンダリング注記: ノード（丸、六角形、矩形）はクリーンなフラットフィルと適度に厚いストロークアウトラインを使用。接続線（エッジ）は流動的で有機的な曲線、または直線ガイドで統一。グラデーションは避け、フラットかつ明確な色面で表現。ノード内の小テキストは可読性を保つ。

タイプ注記: 複数ノードの相互接続ネットワーク図、またはレイヤー別階層図。エージェント・オーケストレーション層がハブ的役割を担い、そこから各専門領域（RAG、推論、UI、評価）へのパイプラインが放射状または流線型に広がる構造が効果的。技術的な統合性と学習パスの複合性を両立させる。

パレット注記: tech-vibrant パレットは生成AI技術の先進性とOSSコミュニティのエネルギーを同時に伝え、ディープサイアン・ネオンマゼンタ・パープルが技術的な信頼感・革新性を、シアン・ピンクがインタラクティブで親しみやすい開発体験を提供。ゴールドアクセントは推奨学習ルート（最短体験パス）を視覚的に強調する。
