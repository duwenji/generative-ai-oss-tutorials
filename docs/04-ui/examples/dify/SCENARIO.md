# Dify シナリオ撮影計画

このファイルは [../02-dify.md](../02-dify.md) の証跡を本文手順に一致させるための台本です。

## シナリオ

### 01. コンテナ起動確認

- 操作: `docker compose ps`
- 採用条件: dify-api 等の主要サービスが `Up`
- 保存: `01-docker-compose-ps.png`

### 02. 初期セットアップ

- 操作: `http://localhost:3000` へアクセスし初期画面を表示
- 採用条件: セットアップ画面の主要入力欄が見える
- 保存: `02-dify-setup.png`

### 03. Provider 設定

- 操作: LLM Provider 設定画面を開く
- 採用条件: Provider 種別と設定欄が読める
- 保存: `03-llm-provider.png`

### 04. アプリ作成

- 操作: Chat アプリを新規作成
- 採用条件: 作成済みアプリ名が画面で確認できる
- 保存: `04-app-created.png`

### 05. 送信前入力

- 操作: `こんにちは。3行で自己紹介して。` を入力して送信前撮影
- 採用条件: 入力文が見える、未送信状態
- 保存: `05-chat-input.png`

### 06. 送信後応答

- 操作: 05 から送信し結果表示後に撮影
- 採用条件: 応答文または追跡可能なエラー文が見える
- 保存: `06-chat-output.png`

### 07. Workflow Builder

- 操作: `Start -> LLM -> End` 構成を作り、ノードをドラッグ移動
- 採用条件: 3 ノードと接続関係が読める
- 保存: `07-workflow-builder.png`

## 品質ゲート

1. 05/06 は必ず別状態
2. 03 は設定画面であること
3. 07 はチャット画面代替でないこと
4. run-log のエラー記録と 06 の内容が一致すること
