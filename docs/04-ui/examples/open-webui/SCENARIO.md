# Open WebUI シナリオ撮影計画

このファイルは [../01-open-webui.md](../01-open-webui.md) の証跡を、本文手順と同じ意味で再取得するための台本です。

## 目的

- 各画像が本文の該当手順を直接証明すること
- 送信前/送信後の状態差を明確に分けること
- 再撮影時に同じ品質で再現できること

## シナリオ

### 01. コンテナ起動確認

- 目的: Open WebUI と Ollama が起動していることを示す
- 操作: `docker compose ps`
- 採用条件: open-webui / ollama が `Up`、ポート表示あり
- 保存: `01-docker-compose-ps.png`

### 02. 初回サインアップ

- 目的: 初回管理者作成画面に到達できることを示す
- 操作: `http://localhost:8080` へアクセス
- 採用条件: 見出しと主要入力欄が読める
- 保存: `02-open-webui-signup.png`

### 03. モデル選択

- 目的: 利用モデルが UI 上で選択されていることを示す
- 操作: モデルセレクタを開いて対象モデルを選択
- 採用条件: 選択モデル名が読める
- 保存: `03-model-selected.png`

### 04. 送信前入力

- 目的: 送信前の入力内容を示す
- 操作: `こんにちは。3行で自己紹介して。` を入力し、送信前に撮影
- 採用条件: 入力文が読める、応答未表示
- 保存: `04-first-chat-input.png`

### 05. 送信後応答

- 目的: モデル応答が返ることを示す
- 操作: 04 から送信し、応答表示後に撮影
- 採用条件: 応答文が読める、04 と別状態
- 保存: `05-first-chat-output.png`

### 06. 履歴確認

- 目的: 会話履歴が保持されることを示す
- 操作: サイドバー履歴を表示して撮影
- 採用条件: 会話タイトルまたは履歴項目が読める
- 保存: `06-history-sidebar.png`

## 品質ゲート

1. 01-06 の意味をそれぞれ 1 文で説明できる
2. 04 は送信前、05 は送信後である
3. ローディング中画面を採用しない
4. run-log と画像の意味が一致している
