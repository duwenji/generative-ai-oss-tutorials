# 教材構造統一 設計ドキュメント

## 目的

`generative-ai-oss-tutorials` 配下の個別教材ファイル（`01-*.md` 〜）を、
`00_STYLE_GUIDE.md` に定める標準見出し順に揃え、ファイル間の一貫性を確保する。

## 背景

- Phase 1/1.5（入口整備・API現代化）は完了済み
- ファイルごとに独自セクションや見出し順のばらつきがあり、読者が迷いやすい
- 今回は Phase 2（標準化）の一環として構造を統一する

---

## 統一後の標準構造

個別教材ファイル（`01-*.md` 〜）は以下の順序で構成する。

この例では、教材構造統一 設計ドキュメント の基本的な利用手順を示します。サンプルコードの意図と、実行時に何が起こるのかを確認しながら読み進めると理解しやすくなります。

```
# {ツール名} - {一行説明}

> 📖 {難易度} | 前提: {前提スキル}

## この教材で身につくこと
- 箇条書き 3〜5 項目

## 概要
- ツールの説明・主な機能
- メリット / デメリット
- バージョン・公式ドキュメントリンク
- ※独自セクション「OSS概要・公式情報」「特徴・できること」「比較・選定ポイント」はここに統合

## 位置づけ
- mermaid 図（flowchart LR 推奨）
- 一言説明

## 実行フロー
- mermaid 図（flowchart TD 推奨）
- 説明文

## 最小セットアップ
- 前提スキル
- 環境要件
- インストール手順（bash コードブロック）
- セキュリティ注意（APIキーを扱う場合）

## 実ソースコード
### Python: {ファイル名}
### JavaScript: {ファイル名}
（言語がない場合は該当言語のみ）

## 演習課題
1. 〜
2. 〜
3. 〜

### 解答の目安
（各問の確認ポイント）

## 理解度チェック
1. 〜
2. 〜
3. 〜

### 解説の要点
（各問の要点）

## 参考リンク
- [公式ドキュメント](...)
- [GitHub Repository](...)

---
[← 前へ](...) | [次へ →](...)
```

### セクション統合マッピング

| 既存セクション名（例） | 統合先 |
|----------------------|--------|
| OSS概要・公式情報 | ## 概要 |
| 特徴・できること | ## 概要 |
| できないこと・制約事項 | ## 概要（メリット/デメリット内） |
| 比較・選定ポイント | ## 概要 末尾 |
| 公式情報の参照・引用について | ## 概要 内バージョン記載に統合 |
| コンセプト / 仕組み | ## 概要 |
| 前提条件 | ## 最小セットアップ |
| サンプル / 検証 | ## 実ソースコード 内に統合 |
| 補足 / よくある質問 | ## 参考リンク の直前に残す（内容が有益な場合） |

## 変換ルール

### 必須対応

| 状態 | 対処 |
|------|------|
| 見出し順がスタイルガイドと異なる | 標準順に並べ替える |
| 独自セクションがある | 内容を最も近い標準セクションに統合 |
| 標準セクションが欠けている | ツールに合わせた最小内容で新規追加 |
| 重複説明がある（「この教材で身につくこと」と「主な機能」が同内容） | 片方を削除・簡潔化 |
| 前後ナビゲーションリンクが欠けている | 追加 |

### 内容変更の範囲

- **構造・順序の変更**: 積極的に行う
- **既存コンテンツの削除**: 重複・冗長な箇所のみ
- **コードサンプルの変更**: 行わない（構造整理のみ）
- **新規コンテンツの追加**: 欠けているセクション（演習課題・理解度チェック）に限定

## カテゴリ完了後の確認チェックリスト

各カテゴリの処理完了後、以下を確認する。

- [ ] 全ファイルで見出し順がスタイルガイド（標準構造）と一致している
- [ ] 「この教材で身につくこと」と他セクションに重複した説明がない
- [ ] mermaid 図が「位置づけ」「実行フロー」の両セクションに存在する
- [ ] 演習課題が 3 問以上ある
- [ ] 理解度チェックが 3 問以上ある
- [ ] 前後ナビゲーションリンクが正しいファイルを指している
- [ ] コードブロックに言語指定がある

## 非対象・スコープ外

- `00-README.md`（カテゴリ入口ページ）の構造変更
- コードサンプルの内容変更・バージョンアップ
- 新規カテゴリ・新規ファイルの追加
- MASTER-INDEX.md / QUICK-REFERENCE.md の変更
