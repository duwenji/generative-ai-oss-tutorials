# Contributing

このリポジトリへの改善提案や修正は歓迎します。

## 基本方針

- 単なる機能紹介ではなく、**OSSどうしの比較観点・選定理由**に結びつく内容を優先してください。
- コマンドや手順は、実際に動かして確認できる内容にしてください。
- バージョン依存の強い内容（API仕様、CLIオプションなど）には確認時点が分かる注記を添えてください。
- 大きな変更は `Issue` で背景を共有してから進めてください。

## 推奨フロー

1. `Issue` を作成する
2. branch を切る
3. 変更を加える
4. `npm run ebook:step1` ～ `ebook:step3`（該当する範囲）を実行する
5. `Pull Request` を作成する

### コマンド補足

- `npm run ebook:step1` / `ebook:step2` / `ebook:step2b` / `ebook:step3`
  - `run`: `package.json` の script を実行する
  - `ebook:stepN`: ebook 出力（EPUB / PDF など）を段階的に生成する script 名
  - **目的:** 公開物の build エラーやリンク崩れを早めに見つける

## レビュー観点

- OSSの比較観点・選定理由が実践的で正確になっているか
- 誤解を生む説明がないか
- コマンドや手順が再現可能か
- バージョン依存の記述が古くなっていないか
